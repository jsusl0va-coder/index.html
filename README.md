Руководство пользователя
1. Назначение программы
Diet Planner — это веб-приложение, которое автоматически составляет недельный план питания с учётом вашего возраста, веса и выбранных приоритетных витаминов. Программа не требует установки и работает в любом современном браузере.
2. Требования к устройству
Для работы приложения подойдёт любой современный браузер: Google Chrome, Mozilla Firefox, Safari или Microsoft Edge. Устройством может быть как компьютер, так и планшет или смартфон — дизайн адаптируется под размер экрана. В браузере должен быть включён JavaScript.
3. Начало работы
Шаг 1. Откройте файл index.html
Дважды щёлкните по файлу `index.html» — откроется главная страница с формой для ввода параметров.
Шаг 2. Заполните ваши данные
В поле «Ваш возраст (лет)» введите целое число от 1 до 120. Пример: 32.
В поле «Ваш вес (кг)» введите число от 20 до 250. Можно использовать десятые доли. Пример: 70.5.
Шаг 3. Выберите важные витамины
Отметьте галочками один или несколько витаминов из списка: A, C, D, E, B12, B6, K, B1. Если ни один витамин не выбран — программа сама подберёт максимально разнообразное и сбалансированное меню.
Шаг 4. Сгенерируйте план
Нажмите на кнопку «Составить недельный план питания →». Система сохранит ваши данные и автоматически перейдёт на страницу с результатом.

4. Работа с планом питания
После перехода на страницу «plan.html» вы увидите несколько информационных блоков.
Верхняя часть страницы содержит ваши параметры: возраст, вес, возрастную группу (ребёнок до 17 лет, взрослый от 18 до 59 лет или пожилой от 60 лет) и список выбранных витаминов.
Основная часть — это недельное меню. Каждый день с понедельника по воскресенье представлен отдельной карточкой. Внутри карточки расписаны четыре приёма пищи:
•	завтрак с указанием блюда, его калорийности и витаминов;
•	перекус с аналогичной детализацией;
•	обед;
•	ужин.
В правом верхнем углу каждой карточки показана общая калорийность дня.
Под каждым приёмом пищи вы найдёте витаминные бейджи (например, 💊 A, 💊 C) — они показывают, какие витамины содержатся в этом блюде.
Внизу каждой карточки расположена жёлтая заметка с тремя типами информации:
•	возрастная рекомендация (например, для пожилых людей — повышенный белок и витамин D);
•	совет по выбранным витаминам;
•	напоминание о том, что калорийность рассчитана исходя из вашего веса.
В самом низу страницы находится блок с персональными рекомендациями по продуктам. Там перечислено, какие конкретные продукты богаты выбранными вами витаминами. Например, для витамина A — морковь, тыква, печень; для витамина C — цитрусы, болгарский перец.
5. Кнопки управления
На странице плана есть две кнопки.
Кнопка «Вернуться к настройкам» — возвращает вас на главную страницу `index.html», чтобы вы могли изменить возраст, вес или набор витаминов и сгенерировать новый план.
Кнопка «Распечатать / Сохранить PDF» — открывает стандартное окно печати браузера. В этом окне можно выбрать принтер или опцию «Сохранить как PDF», чтобы сохранить недельный план на компьютер.

6. Частые вопросы
Почему план не сгенерировался?
Убедитесь, что возраст и вес введены корректно (возраст от 1 до 120, вес от 20 до 250). Если вы перешли на страницу plan.html» напрямую, без заполнения формы, программа автоматически перенаправит вас обратно наindex.html».
Можно ли изменить витамины после генерации?
Да. Нажмите «Вернуться к настройкам», измените выбор витаминов и снова нажмите «Составить план». Старый план будет заменён новым.
Почему в некоторые дни повторяются одни и те же блюда?
Алгоритм подбирает блюда, которые максимально богаты выбранными витаминами. Если вы выбрали очень узкий набор витаминов (например, только витамин D), подходящих блюд может быть немного, и они будут повторяться. Это не ошибка, а особенность работы подбора.
Учитывает ли план аллергии или заболевания?
Нет. Программа даёт общие рекомендации для здорового человека. При наличии пищевой аллергии, хронических заболеваний или особых диет обязательно проконсультируйтесь с врачом или диетологом.








Руководство разработчика
1. Архитектура приложения
Проект представляет собой статическое веб-приложение, состоящее из двух HTML-страниц. Каждая страница содержит встроенные стили CSS и встроенный JavaScript. Приложение не требует серверной части и может работать полностью локально на компьютере пользователя.
Структура проекта:
•	index.html — страница с формой для ввода возраста, веса и выбора витаминов.
•	plan.html — страница с генерацией и отображением недельного плана питания.
Хранение данных: Для передачи данных между страницами используется механизм localStorage. Ключ, по которому сохраняются данные, называется dietPlannerData.
Передача данных: При отправке формы на index.html» данные сохраняются вlocalStorage, после чего происходит переход наplan.html. Страницаplan.html» считывает данные из localStorage, генерирует план и отображает его. Если данных нет (пользователь зашёл на plan.html» напрямую), выполняется автоматический редирект наindex.html».

2. Структура файла index.html
HTML-разметка содержит форму с тремя основными блоками: поле ввода возраста, поле ввода веса и блок с чекбоксами витаминов.
JavaScript в index.html выполняет следующие задачи:
•	Рендеринг списка витаминов. Массив VITAMINS_LIST содержит объекты с идентификаторами, названиями и значениями витаминов. Функция renderVitamins() динамически создаёт чекбоксы и подписи к ним.
•	Обработка отправки формы. Функция handleFormSubmit(event):
o	предотвращает стандартную отправку формы;
o	считывает значения возраста и веса;
o	проверяет, что возраст находится в диапазоне от 1 до 120, а вес — от 20 до 250;
o	собирает выбранные витамины из отмеченных чекбоксов;
o	формирует объект userData и сохраняет его в localStorage;
o	перенаправляет пользователя на plan.html.
•	Восстановление ранее введённых данных. При загрузке страницы код проверяет, есть ли сохранённые данные в localStorage. Если есть — поля формы автоматически заполняются, а нужные чекбоксы отмечаются.

3. Структура файла plan.html
HTML-разметка содержит контейнеры для отображения информации о пользователе, выбранных витаминах, недельного плана, а также кнопки управления.
Константа базы блюд MEAL_DB — это объект, разделённый на четыре категории: breakfast (завтраки), lunch (обеды), dinner (ужины), snack (перекусы). Каждое блюдо представляет собой объект с четырьмя полями:
•	name — название блюда;
•	vitamins — массив строк с обозначениями витаминов (например, ["A","C","B1"]);
•	kcal — калорийность блюда;
•	desc — дополнительное описание или рекомендация.
Функции расчёта и логики:
•	getAgeGroup(age) — определяет возрастную группу. Возвращает "child" для возраста до 17 лет, "adult" для возраста от 18 до 59 лет, "senior" для возраста 60 лет и старше.
•	calculateDailyCalories(weight, ageGroup) — рассчитывает суточную норму калорий. Базовая формула: вес умножается на 28. Для детей коэффициент увеличивается до 32, для пожилых — уменьшается до 25. Результат ограничивается диапазоном от 1500 до 2800 калорий.
•	getMealKcalRange(dailyKcal) — распределяет суточную калорийность по приёмам пищи. Завтрак получает 25% калорий, обед — 35%, ужин — 25%, перекус — 15%. Функция возвращает объект с округлёнными значениями.
•	pickBestMeal(mealArray, targetVitamins) — выбирает лучшее блюдо из массива на основе целевых витаминов. Алгоритм вычисляет оценку: за каждый целевой витамин, присутствующий в блюде, начисляется 2 балла; дополнительно блюдо получает 0.5 балла за каждый свой витамин (поощрение за разнообразие). Выбирается блюдо с максимальной оценкой. Если список целевых витаминов пуст, выбор основывается только на разнообразии витаминов в блюде.
•	generateDayMenu(selectedVitamins, targetKcalRanges) — создаёт меню на один день. Для каждого из четырёх приёмов пищи вызывается pickBestMeal(), после чего результат форматируется в удобный для отображения вид.
•	generateWeeklyPlan(selectedVitamins, dailyTargetKcal, ageGroup, weight) — генерирует план на 7 дней. Функция создаёт массив дней недели (с понедельника по воскресенье) и для каждого дня вызывает generateDayMenu(). К каждому дню также добавляются текстовые заметки: возрастная рекомендация, витаминный совет и напоминание о весе.
•	renderPlan(plan, selectedVitamins, age, weight, ageGroup, dailyKcal) — отрисовывает план в DOM. Функция очищает контейнер, затем для каждого дня создаёт карточку с заголовком, приёмами пищи, витаминными бейджами и заметкой. В конце добавляется общий блок с рекомендациями по продуктам для выбранных витаминов.
•	loadAndGeneratePlan() — главная функция, которая вызывается при загрузке plan.html. Она считывает данные из localStorage, определяет возрастную группу, рассчитывает калорийность, генерирует недельный план и отображает его. Если данные отсутствуют, выполняется редирект на index.html.
Обработчики событий:
•	кнопка «Вернуться к настройкам» вызывает переход на index.html;
•	кнопка «Распечатать» вызывает встроенную функцию браузера window.print().

4. Структура данных
Формат userData в localStorage:
Объект содержит четыре поля: age (возраст, число), weight (вес, число), vitamins (массив строк с обозначениями витаминов, например ["A","C","D"]) и timestamp (метка времени для возможного использования в будущем).
Формат блюда в MEAL_DB:
Каждое блюдо — это объект с полями name (строка), vitamins (массив строк), kcal (число), desc (строка с дополнительной информацией).
Формат дня в недельном плане:
Объект содержит день недели (day), объект meals с четырьмя приёмами пищи (каждый содержит название, калории, витаминные бейджи, описание и ожидаемую калорийность), числовое поле totalCalories, а также строковые поля ageNote, vitAdvice и weightNote для дополнительных заметок.

5. Стилизация (CSS)
Основные принципы оформления:
Фон страницы выполнен в виде градиента от сиреневого к фиолетовому. Основные карточки (контейнеры с планом) имеют белый полупрозрачный фон, скруглённые углы (от 28 до 40 пикселей) и тень для создания глубины.
Ключевые CSS-классы:
•	.card — основная карточка с большим скруглением, белым фоном и тенью.
•	.day-card — карточка отдельного дня, имеет белую заливку, тонкую рамку и лёгкую тень.
•	.day-header — верхняя часть карточки дня с названием дня и калорийностью. Фон — светло-серый.
•	.meal — блок одного приёма пищи, использует flexbox для расположения времени приёма и описания.
•	.vitamin-highlight — бейдж для отображения витаминов (фиолетовый фон, маленькие буквы).
•	.extra-note — жёлтая заметка внизу карточки дня с дополнительными рекомендациями.
Адаптивность:
Для устройств с шириной экрана до 680 пикселей применяются специальные правила: заголовок дня выстраивается вертикально, а блоки приёмов пищи также переключаются в вертикальный режим, чтобы удобно читать на мобильных устройствах.
Печать:
Используется стандартный механизм window.print(). Браузер сам определяет, как отображать страницу при печати. Фоновые градиенты при печати обычно не сохраняются, но текст и таблицы остаются читаемыми.

6. Алгоритм выбора блюда (подробно)
Функция pickBestMeal является сердцем генератора. Она получает массив блюд (например, все завтраки) и список целевых витаминов, выбранных пользователем.
Алгоритм пошагово:
1.	Если массив блюд пуст, возвращается undefined (но в реальности база заполнена).
2.	Переменной best присваивается первое блюдо, а bestScore — значение -1.
3.	Для каждого блюда вычисляется оценка score, которая изначально равна 0.
4.	Если список целевых витаминов не пуст, то для каждого целевого витамина проверяется, есть ли он в массиве vitamins блюда. За каждое совпадение добавляется 2 балла.
5.	Дополнительно к оценке прибавляется 0.5 балла за каждый витамин, который есть в блюде (стимулируется выбор блюд с богатым витаминным составом).
6.	Если список целевых витаминов пуст, то оценка вычисляется просто как длина массива vitamins блюда, умноженная на 2.
7.	Если текущая оценка больше bestScore, то блюдо становится лучшим.
8.	В конце функция возвращает лучшее блюдо.
Этот алгоритм гарантирует, что при выбранных витаминах программа выберет блюдо, которое покрывает их максимально, а при отсутствии выбора — самое витаминно-разнообразное.

7. Расчёт калорийности (подробно)
Функция calculateDailyCalories вычисляет суточную норму калорий следующим образом:
Базовая формула: вес × 28. Это среднее значение для взрослого человека с умеренной активностью.
Для детей (до 18 лет) используется коэффициент 32, так как дети обычно более активны и находятся в фазе роста.
Для пожилых людей (от 60 лет) используется коэффициент 25, так как метаболизм с возрастом замедляется.
После умножения результат проверяется на границы: минимальная калорийность не может быть ниже 1500 ккал (безопасный минимум), максимальная — не выше 2800 ккал (избыток для большинства людей).
Полученное значение округляется до целого числа.
Функция getMealKcalRange распределяет эти калории по приёмам: завтрак — 25%, обед — 35%, ужин — 25%, перекус — 15%. Проценты пересчитываются в абсолютные значения и также округляются.

8. Расширение и модификация
Добавление нового блюда:
В файле plan.html найдите объект MEAL_DB. В нужную категорию (например, breakfast) добавьте новый объект с полями name, vitamins, kcal, desc. Пример:
text
breakfast: [
  ...,
  { name: "Новая каша", vitamins: ["B1","B6"], kcal: 300, desc: "описание" }
]
Добавление нового витамина:
В файле index.html добавьте новый объект в массив VITAMINS_LIST. В файле plan.html в блюдах, которые содержат этот витамин, добавьте его обозначение в массив vitamins. Также в функции renderPlan дополните блок vitaminRichFoods описанием продуктов для нового витамина.
Изменение формулы калорий:
Отредактируйте функцию calculateDailyCalories в файле plan.html. Измените коэффициенты (28, 32, 25) или добавьте новые условия.
Изменение распределения калорий по приёмам:
Отредактируйте функцию getMealKcalRange. Проценты задаются внутри неё числами 0.25, 0.35, 0.25, 0.15.
Добавление нового возрастной группы:
В функцию getAgeGroup добавьте новое условие. Например, для группы «подростки» можно добавить диапазон 14–17 лет. Затем в функции calculateDailyCalories добавьте соответствующий коэффициент.

9. Возможные ошибки и их обработка
Ситуация: пользователь ввёл возраст меньше 1 или больше 120 на странице index.html.
Обработка: функция handleFormSubmit вызывает alert() с сообщением «Пожалуйста, укажите корректный возраст (от 1 до 120 лет)». Отправка формы прерывается, пользователь остаётся на той же странице.
Ситуация: пользователь ввёл вес меньше 20 или больше 250.
Обработка: аналогично вызывается alert() с сообщением о весе.
Ситуация: пользователь перешёл на plan.html напрямую, без заполнения формы (например, сохранил ссылку в закладки).
Обработка: функция loadAndGeneratePlan проверяет наличие данных в localStorage. Если данных нет, выполняется команда window.location.href = 'index.html', и пользователь автоматически перенаправляется на главную страницу.
Ситуация: массив выбранных витаминов пуст.
Обработка: это штатная ситуация. Функция pickBestMeal работает в режиме «без целевых витаминов» и выбирает блюдо с максимальным разнообразием.
Ситуация: в базе MEAL_DB нет блюд для какой-то категории (например, удалены все завтраки).
Обработка: функция pickBestMeal получит пустой массив и вернёт undefined. На практике база всегда заполнена, но для надёжности можно добавить проверку и возвращать блюдо-заглушку.

10. Зависимости и требования к окружению
Приложение использует только чистый HTML, CSS и JavaScript. Никаких внешних библиотек или фреймворков не требуется. Шрифты используются системные (Segoe UI, Tahoma, Verdana) — они есть во всех современных операционных системах. JavaScript должен быть включён в браузере. Приложение работает локально без интернета (после того как страницы открыты).

11. Рекомендации по доработке
Добавление исключения продуктов (учёт аллергий): задача средней сложности. Потребуется добавить новое поле в форму на index.html (например, список аллергенов), сохранять его в localStorage, а в функции pickBestMeal фильтровать блюда, содержащие исключённые продукты. Для этого каждое блюдо должно получить дополнительное поле allergens.
Расширение базы блюд: задача низкой сложности. Достаточно просто добавлять новые объекты в MEAL_DB. Чем больше блюд, тем разнообразнее будет недельный план.
Учёт пола пользователя: задача низкой сложности. Добавить поле «Пол» на index.html, сохранять его, а в формуле расчёта калорий использовать разные коэффициенты для мужчин и женщин (например, для мужчин базовый коэффициент 30, для женщин — 25).
Сохранение нескольких планов: задача высокой сложности. Потребуется изменить структуру хранения с одного объекта на массив планов, добавить интерфейс для переключения между ними и, возможно, использовать IndexedDB вместо localStorage из-за ограничения на объём.
Экспорт в PDF с сохранением стилей: задача средней сложности. Вместо window.print() можно подключить библиотеку html2pdf.js, которая позволяет точно сохранить все стили и цвета в PDF-файл.

12. Заключение
Проект «Персональный план питания — Diet Planner» представляет собой полностью рабочее веб-приложение на чистом JavaScript с понятной модульной архитектурой. Код легко читается, модифицируется и не требует развёртывания на сервере. Приложение может использоваться как есть или служить основой для более сложной системы диетологического консультирования.



8. Ограничения программы
План питания составлен для здорового человека без специальных диетических требований. Калорийность рассчитана приблизительно по усреднённым формулам. Приложение не заменяет профессиональную консультацию диетолога.
